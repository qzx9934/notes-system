# 资料整理工作流程（资料 → 规范笔记）

把零散的工作资料（文本、Word、Markdown、PDF、PPTX 等）标准化地整理成本系统的结构化笔记。本文给出**一套通用流程**和**三种落地方式**，并说明用到的 AI 工具、是否建 Skill，以及手机端 PWA 安装。

> 接口细节见 [`API文档.md`](../API文档.md)；提示词见 [`笔记整理提示词.md`](笔记整理提示词.md)。

---

## 0. 总览：四步流水线

```
①抽取文本  →  ②AI整理成规范JSON  →  ③API批量上传(去重合并)  →  ④验证与回填
 各种格式      章节归类+字段规范        /api/notes/ingest          看 added/merged/skipped
```

核心思想：**不手敲笔记**。让 AI 把资料整理成符合字段规范的 JSON，再用 `ingest` 接口一次性入库；重复标题自动合并更新，可反复增量上传。

---

## 1. 一次性准备

笔记上传到**已部署的公网服务器**。准备一个管理员 API 令牌：

1. 浏览器登录系统 → 「令牌管理」→ 新建令牌（角色 admin）→ 复制明文（**只显示一次**）。
2. 在你做整理的机器上设置环境变量（写进 `~/.bashrc` / `~/.zshrc` 持久化）：

   ```bash
   export NOTES_API_URL=https://你的域名
   export NOTES_API_TOKEN=ntk_你的令牌
   ```

> 令牌等同管理员写权限：不要提交进仓库、不要贴进聊天/截图。泄露了就去「令牌管理」吊销重建。

自检：

```bash
curl -s -o /dev/null -w "%{http_code}\n" -H "X-API-Token: $NOTES_API_TOKEN" "$NOTES_API_URL/api/stats"
# 200 = 令牌可用
```

## 2. 三种落地方式（按你手头的工具选）

### 方式 A · Claude Code / Codex CLI（推荐，已内置 Skill）

最省事：仓库里已建好 Skill **`organize-notes`**（见 `.claude/skills/organize-notes/`）。在仓库目录开 Claude Code，直接说：

```
把 ~/资料/汽机超速保护.pdf 和 培训.pptx 整理成笔记上传
```

或显式触发：`/organize-notes ~/资料/*.pdf`

Skill 会自动完成：抽取文本 → 按章节整理成 JSON → 调 `ingest` 上传 → 回报 added/merged/skipped。
依赖你已 `export` 好 `NOTES_API_URL` / `NOTES_API_TOKEN`。

> Codex CLI 同理：把 `.claude/skills/organize-notes/SKILL.md` 的内容作为任务说明喂给它即可，流程一致。

### 方式 B · 桌面聊天客户端（粘贴 / 上传文件，可建专属智能体）

适合不开命令行、直接在聊天窗口处理。

#### B-1 通用：Claude 桌面 / ChatGPT 等

1. 打开 [`笔记整理提示词.md`](笔记整理提示词.md)，复制【提示词正文】。
2. 连同资料（粘贴文本，或上传 Word/PDF/PPTX——这些工具大多能直接读文件）一起发给模型。
3. 模型输出 JSON → 存成 `notes.json` → 上传：`python note_cli.py import notes.json`（或交给 Claude Code 用 `/organize-notes` 上传）。

#### B-2 推荐：Cherry Studio（创建"笔记整理"智能体，一次配置长期复用）

Cherry Studio 是跨平台桌面客户端，可对接 Claude / GPT / Gemini / 本地 Ollama 等模型，并支持**自建智能体(助手)**、**知识库(RAG)**、**文件上传**与 **MCP 工具**。把整理规则固化成一个智能体后，以后只需把资料丢进去，无需每次重贴提示词。

**① 创建智能体（一次性）**

1. 左侧「智能体 / 助手」→「添加 / 创建」。
2. 名称：`电厂笔记整理助手`。
3. **系统提示词**：粘贴 [`笔记整理提示词.md`](笔记整理提示词.md) 的【提示词正文】（含 26 章节编码与字段规范）。
4. **模型**：选支持读文件、上下文较大的（Claude / GPT-4o / Gemini 等，按你已配好的 API Key）。
5. **参数**：温度调低（0~0.3）保证稳定吐出规范 JSON；如有"联网/自由发挥"开关建议关掉。
6. 保存。也可把"来源固定为某值""日期默认今天"等个人习惯一起写进系统提示，长期风格一致。

**② 用它整理资料**

- 新建对话选「电厂笔记整理助手」→ 把 Word / PDF / PPTX / 文本拖进输入框发送，智能体按系统提示输出 JSON 数组。
- 资料很多 / 很长：先建一个**知识库**把这批文件导入，再让智能体"基于知识库整理成笔记 JSON"，适合一次性消化整套规程或培训材料。
- 若它输出夹带解释文字，回一句"只输出 JSON 数组本身"即可。

**③ 上传到系统**

- 复制 JSON 存成 `notes.json` → `python note_cli.py import notes.json`（上传到你的公网地址，自动去重合并）。
- 进阶（可选）：Cherry Studio 支持 **MCP**，可自建一个调用本系统 `/api/notes/ingest` 的 MCP 工具，让智能体整理完**直接入库**、免去手动上传；没精力做就用上面的 `note_cli` 方式。

### 方式 C · API 脚本自动化（批量、无人值守，后续再上）

把"读文件夹 → 调大模型 → ingest"串成脚本，定时处理收件箱式目录。骨架见 `API文档.md` 第七节的 Python 示例 + 本仓库 `extract_text.py` 抽取函数。等前两种方式跑顺了再做。

## 3. 资料抽取对照表（第①步）

| 格式 | 怎么抽取 |
| --- | --- |
| 纯文本 / 粘贴 | 直接用 |
| `.md` `.txt` `.csv` `.json` | Read 工具，或 `python .claude/skills/organize-notes/extract_text.py 文件` |
| `.docx` | `python .claude/skills/organize-notes/extract_text.py x.docx`（**零依赖**，离线可用） |
| `.pptx` | 同上，按幻灯片顺序输出 |
| `.pdf` | Claude Code 用 Read 直接读最省事；批量用 `extract_text.py`（需 `pdftotext`/`pypdf`/`pymupdf` 之一） |
| `.doc` `.ppt`（旧版） | 先另存为 `.docx` / `.pptx` |
| 多个文件 | 一次传多个路径，脚本按文件分段 |

`extract_text.py` 用法：

```bash
python .claude/skills/organize-notes/extract_text.py a.docx b.pptx c.pdf > 原始文本.txt
```

## 4. 字段与章节规范（第②步）

每条笔记是一个 JSON 对象。完整字段、校验规则与 26 个章节编码见 [`API文档.md`](../API文档.md) 第二、三节。要点：

- `section`（必填）+ `title`（必填）；`code` 编号由后端自动生成。
- `level` 只能是 `★`/`★★`/`★★★`；涉及保护/跳闸/安全取 `★★★`。
- `date` 须 `YYYY-MM-DD`。
- 批量接口对非法 `level`/`date` 会**自动回退**默认值；但 `section` 非法会被**跳过**，所以章节一定要选对。

整理质量要求：**一条笔记一个要点**、标题能独立看懂、**数字/定值/动作条件原样保留**、不杜撰原文没有的内容。

## 5. 上传与去重（第③步）

```bash
python note_cli.py import notes.json          # 推荐
# 或
curl -s -X POST "$NOTES_API_URL/api/notes/ingest" \
  -H "Content-Type: application/json" -H "X-API-Token: $NOTES_API_TOKEN" \
  --data-binary @notes.json
```

`ingest` 按「章节 + 标题」去重：命中已有条目则**更新**而非新增。因此可以反复增量上传、修订同一批笔记，不会产生重复。

---

## 6. 验证与排错（第④步）

响应里三个计数：

- `added` 新建 / `merged` 命中去重被更新 / `skipped` 跳过（`skipped_list` 给原因）。
- 常见 `skipped` 原因：`section` 非法、`title` 为空 → 修正后重传。

抽查：

```bash
python note_cli.py search 超速保护
python note_cli.py stats
```

| 现象 | 处理 |
| --- | --- |
| `401` | 令牌缺失/失效 → 检查 `NOTES_API_TOKEN`，或重建令牌 |
| `403` | 令牌是 viewer 角色 → 用 admin 令牌 |
| `skipped` 多 | 多为章节选错 → 核对编码表 / `GET /api/sections` |
| JSON 报错 | `python -m json.tool notes.json` 定位语法问题 |

## 7. 是否需要建 Skill？——已建好

**需要，且已经建好。** 因为你主要用 Claude Code / Codex CLI，Skill 能把"抽取→整理→上传→回报"固化成一句话触发的标准动作，避免每次重复贴提示词。

- 位置：`.claude/skills/organize-notes/`
  - `SKILL.md` —— 触发说明与完整步骤（含章节表、字段规范、上传命令）
  - `extract_text.py` —— 零依赖文本抽取器（docx/pptx/txt/md；pdf 多后端）
- 触发：在仓库内开 Claude Code，自然语言说"整理成笔记上传"，或 `/organize-notes <文件>`。
- 维护：章节体系或字段规则变了，只改 `SKILL.md` 与 `API文档.md`，流程不变。

> 想新增个人专用 Skill（比如固定某种资料的整理风格），复制 `organize-notes` 目录改名即可；放 `.claude/skills/` 下随仓库走，放 `~/.claude/skills/` 下则全局可用。

## 8. 手机端：安装为 App（PWA）

本系统已做成 **PWA**，无需上架商店即可"装"到手机主屏，像原生 App 一样全屏使用：

- **iOS Safari**：打开站点 → 分享 → 「添加到主屏幕」。
- **Android Chrome**：打开站点 → 菜单 → 「安装应用 / 添加到主屏幕」。

安装后有独立图标、全屏显示、二次打开秒开（应用外壳已被 Service Worker 缓存）。
注意：数据仍需联网从服务器读取；离线只能打开外壳，无法登录/查数据。PWA 要求 HTTPS（你的公网部署已满足）。

> 若想要消息推送、调用系统分享/文件等更深的原生能力，再考虑原生 App（Flutter/React Native）；对"手机上随时查阅录入"这个目标，PWA 的成本收益比最高。
