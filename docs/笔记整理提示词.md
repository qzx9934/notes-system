# 笔记整理提示词模板

在 **Claude 桌面 / Cherry Studio / WorkBuddy / ChatGPT** 等聊天工具里，上传或粘贴资料后，把下面整段作为提示词发给模型。模型会输出可直接上传的 JSON。

> 用法：复制【提示词正文】→ 连同你的资料（文本，或上传的 Word/PDF/PPTX）一起发送 → 把模型返回的 JSON 存成 `notes.json`。

---

## 提示词正文（复制这一段）

```
你是电厂运行笔记整理助手。请把我提供的资料整理成结构化笔记，输出一个 JSON 数组，
不要任何解释或 Markdown 代码块标记，只输出 JSON 本身。每个元素是一条笔记，字段如下：

- section：章节编码，必须从下列编码中选最贴切的一个：
  A01锅炉及辅助系统 A02汽轮机及辅助系统 A03电气系统 A04热控自动化系统 A05辅机系统
  A06脱硫脱硝环保系统 A07化水处理系统 A08燃料供应系统 A09除灰除尘系统
  B01机组启停操作 B02正常运行调整 B03定期工作与试验 B04设备切换操作 B05停送电操作
  C01安措与两票管理 C02事故预案与应急处理 C03异常工况判断与处理 C04安全规程与制度
  D01设备参数与运行限额 D02保护定值与联锁逻辑 D03运行规程要点 D04检修质量标准
  E01事故通报与经验反馈 E02培训与考试笔记 E03值班管理 E04技术改造与优化
- title：要点标题，简洁准确，不超过 30 字，能独立看懂
- content：内容详情，可用要点编号；务必原样保留关键数字、定值、动作条件，不要杜撰
- tags：3~5 个关键词，用英文逗号分隔
- level：重要等级，★/★★/★★★；涉及保护、跳闸、安全的取 ★★★
- source：来源，从 规程/培训/工作票/事故预案/事故通报/经验反馈/技术文件/个人总结 中选择
- date：可选，YYYY-MM-DD，缺省即留空

整理原则：一条笔记只讲一个要点；把长文拆成多条；忠于原文，缺失的信息不要编。
```

---

## 拿到 JSON 之后

两种上传方式，任选其一：

1. **命令行**（推荐，可去重合并）：存成 `notes.json`，运行
   ```bash
   export NOTES_API_URL=https://你的域名
   export NOTES_API_TOKEN=ntk_你的令牌
   python note_cli.py import notes.json
   ```
2. **网页端**：登录系统，用「批量录入」粘贴 JSON（如界面提供）；或把 JSON 交给 Claude Code 用 `/organize-notes` 上传。

校验 JSON 是否合法：`python -m json.tool notes.json >/dev/null && echo OK`
