# 宝塔面板部署指南（Supervisor + Gunicorn + Nginx）

本文记录在云服务器（公网 IP + 域名 + HTTPS）上，用**宝塔面板**的 Supervisor 管理器部署本系统的完整步骤与排错经验。

- 示例环境：OpenCloudOS / CentOS 系，宝塔面板，项目目录 `/opt/notes-system`
- 架构：浏览器 →(HTTPS) Nginx 反向代理 →(HTTP) Gunicorn(127.0.0.1:5000) → Flask 应用

> 提示：`deploy/` 目录已被 `.gitignore` 忽略，部署相关配置（`gunicorn_config.py` 等）按设计**放在服务器本地**，不进仓库。本文的配置以示例形式给出，照抄到服务器即可。

---

## 1. 前置条件

- 已安装宝塔面板，并在「软件商店」安装 **Supervisor 管理器**、**Nginx**。
- 已为域名配置好 **HTTPS 证书**（宝塔「网站」里一键申请 Let's Encrypt 即可）。
- 服务器可访问 GitHub（或用其它方式把代码放到 `/opt/notes-system`）。

## 2. 拉取代码、创建虚拟环境、安装依赖

```bash
cd /opt/notes-system
git pull origin master          # 首次则 git clone <仓库> /opt/notes-system

# 创建虚拟环境并装依赖（requirements.txt 已含 gunicorn）
python3 -m venv venv
venv/bin/python -m pip install --upgrade pip
venv/bin/python -m pip install -r requirements.txt
venv/bin/gunicorn --version     # 能打印版本即 OK
```

> 若 `venv/bin/pip` 报 `Permission denied`：改用 `venv/bin/python -m pip ...`；仍不行则 `chmod +x venv/bin/*`；再不行 `rm -rf venv` 后重建。

## 3. Gunicorn 配置

新建 `/opt/notes-system/deploy/gunicorn_config.py`。**安全环境变量通过 `raw_env` 注入**——这样只要「重启」程序就生效，无需 supervisor reread：

```python
# -*- coding: utf-8 -*-
"""Gunicorn 生产配置（supervisor 管理）"""
import multiprocessing

bind = "127.0.0.1:5000"
workers = min(multiprocessing.cpu_count(), 4)
worker_class = "sync"
threads = 2
timeout = 120
graceful_timeout = 30

# 日志交给 supervisor 收集（标准输出/错误），避免依赖 /var/log 目录是否存在
accesslog = "-"
errorlog = "-"
loglevel = "info"

proc_name = "notes-system"
daemon = False

# 启动时注入环境变量（密钥仅存于此服务器本地文件，不入库）
raw_env = [
    "NOTES_HTTPS=1",
    "NOTES_TRUST_PROXY=1",
    "NOTES_SECRET_KEY=__在这里填入强随机串__",
]
```

生成强随机密钥：

```bash
/opt/notes-system/venv/bin/python -c "import secrets; print(secrets.token_hex(32))"
```

把输出替换上面的 `__在这里填入强随机串__`（**不要带尖括号、不要把密钥发到任何聊天/工单里**）。

> 不要写 `accesslog/errorlog = "/var/log/notes-system/..."` 这类路径——若目录不存在，gunicorn 打不开日志会**直接启动失败**。用 `"-"` 最稳。

## 4. Supervisor 程序配置

在宝塔「Supervisor 管理器」新增程序，或直接编辑其配置（示例）：

```ini
[program:notes-system]
command=/opt/notes-system/venv/bin/gunicorn -c /opt/notes-system/deploy/gunicorn_config.py wsgi:app
directory=/opt/notes-system
autorestart=true
startsecs=3
startretries=3
stdout_logfile=/www/server/panel/plugin/supervisor/log/notes-system.out.log
stderr_logfile=/www/server/panel/plugin/supervisor/log/notes-system.err.log
stdout_logfile_maxbytes=2MB
stderr_logfile_maxbytes=2MB
user=root
priority=999
numprocs=1
process_name=%(program_name)s_%(process_num)02d
```

> 环境变量已放在 `gunicorn_config.py` 的 `raw_env` 里，所以这里**可以不写** `environment=`。
> 若改用 supervisor 的 `environment=`，记得它必须是**单独一整行**：`environment=NOTES_HTTPS="1",NOTES_TRUST_PROXY="1",NOTES_SECRET_KEY="..."`，且改后需 reread+update 才生效。

## 5. Nginx 反向代理要点

用宝塔「网站 → 反向代理」指向 `http://127.0.0.1:5000` 即可。确认转发了以下头（宝塔默认模板一般已包含），否则 `NOTES_TRUST_PROXY=1` 拿不到真实客户端 IP：

```nginx
proxy_set_header Host              $host;
proxy_set_header X-Real-IP         $remote_addr;
proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
proxy_set_header X-Forwarded-Proto $scheme;
```

TLS 在 Nginx 层终止，后端只跑 HTTP（`127.0.0.1:5000`）。

## 6. 环境变量说明

| 变量 | 作用 | 公网建议 |
| --- | --- | --- |
| `NOTES_HTTPS` | 经 HTTPS 提供时设 `1`：下发 Secure Cookie + HSTS | `1`（须先配好证书） |
| `NOTES_TRUST_PROXY` | 反向代理之后设 `1`：按 `X-Forwarded-*` 还原真实 IP（登录限流依赖） | `1` |
| `NOTES_SECRET_KEY` | 会话签名密钥（不设会自动生成并存 `.secret_key`） | 设强随机串 |
| `NOTES_CORS_ORIGINS` | 允许跨域来源（逗号分隔）；不设则不开放跨域 | 一般留空 |
| `NOTES_COOKIE_SAMESITE` | 会话 Cookie SameSite（防 CSRF） | 默认 `Lax` |
| `NOTES_API_TOKEN` | 主 API 令牌（命令行/大模型上传） | 按需 |
| `NOTES_DB_PATH` | 数据库路径覆盖 | 一般留空（默认 `backend/notes.db`） |

## 7. 启动 / 重启 / 看日志

- **重启程序（最常用）**：宝塔 → Supervisor 管理器 → `notes-system` → 「重启」。重启会让 gunicorn 重新读取 `gunicorn_config.py`，`raw_env` 随之生效。
- `supervisorctl` 不在系统 PATH 属正常（宝塔的 supervisor 跑在自身 Python 环境）。需命令行时先定位：
  ```bash
  find / -name supervisorctl 2>/dev/null | head
  ```
- 日志：
  ```bash
  tail -n 50 /www/server/panel/plugin/supervisor/log/notes-system.err.log
  tail -n 50 /www/server/panel/plugin/supervisor/log/notes-system.out.log
  ```

## 8. 验证

```bash
# 应用是否起来（401 = 未登录的预期响应）
curl -s -o /dev/null -w "后端: %{http_code}\n" http://127.0.0.1:5000/api/check-auth

# 加固是否生效（应看到 HSTS）
curl -s -D - -o /dev/null http://127.0.0.1:5000/api/check-auth | grep -i strict-transport
# 期望：Strict-Transport-Security: max-age=31536000; includeSubDomains
```

浏览器开域名（HTTPS）→ F12 → Application → Cookies → 看 `session` 的 **Secure** 是否打勾。

> 注意：直连本地 `127.0.0.1:5000` 是 HTTP，看不到 Secure Cookie 生效；但能看到 HSTS 头就说明 `NOTES_HTTPS=1` 已读到。`raw_env` 注入的变量**不会**出现在 `/proc/PID/environ`（那只反映启动时的初始环境），以**响应头**为准。

## 9. 上线检查清单

1. ☐ HTTPS 证书已配好，域名可正常 https 访问
2. ☐ `gunicorn_config.py` 的 `raw_env` 含 `NOTES_HTTPS=1`、`NOTES_TRUST_PROXY=1`、**新生成的** `NOTES_SECRET_KEY`
3. ☐ Nginx 反代转发了 `X-Forwarded-For` / `X-Forwarded-Proto`
4. ☐ 重启后 `curl` 能看到 `401` + `Strict-Transport-Security`
5. ☐ 浏览器登录后**立刻把默认口令 `admin/admin123` 改掉**

## 10. 常见问题（FAQ）

- **`venv/bin/pip: Permission denied`** → 用 `venv/bin/python -m pip ...`；或 `chmod +x venv/bin/*`；或 `rm -rf venv` 重建。
- **`supervisorctl: command not found`** → 正常，用宝塔面板按钮重启；或 `find / -name supervisorctl`。
- **进程启动异常 / 一直重启** → 多半是 `gunicorn_config.py` 里 `accesslog/errorlog` 指向了不存在的目录。改成 `"-"`，或 `mkdir -p` 出那个目录。真实报错看 `notes-system.err.log`。
- **`curl` 没有 HSTS 头** → `NOTES_HTTPS` 没被读到。确认 `raw_env` 写对、并**重启**程序（不是 reload）；以响应头为准排查。
- **会话总是掉线 / 多 worker 不一致** → 确保设置了固定的 `NOTES_SECRET_KEY`（或保留自动生成的 `.secret_key` 文件，不要每次部署删它）。
- **密钥不慎泄露**（贴到聊天/截图/工单等）→ 立即用 `secrets.token_hex(32)` 重新生成、替换并重启，旧密钥即作废。

## 11. 日常维护

```bash
cd /opt/notes-system
git pull origin master
# 然后宝塔 Supervisor 管理器点「重启」
```

数据库 `backend/notes.db` 不受代码更新影响；重启不会丢数据。建议定期备份该文件。
